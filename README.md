# GravityMusic (Музыка Гравитации)

Современный и стильный музыкальный плеер для Android с поддержкой локальных файлов и интеграцией с **Яндекс.Диском**. Приложение написано на Kotlin с использованием Jetpack Compose и Material 3.

## Особенности
* 🎨 **Glassmorphism & Material You:** Поддержка светлой и темной тем, а также динамических цветов Android 12+.
* ☁️ **Интеграция с Яндекс Диском:** Авторизация через OAuth2 и возможность загрузки музыки напрямую из ваших облачных папок.
* 🎧 **Продвинутый плеер:** Поддержка эффектов, A-B повтора, и аппаратного эквалайзера.
* 📋 **Очередь воспроизведения:** Drag-and-Drop (перетаскивание) треков для изменения порядка воспроизведения.
* 📱 **Виджет и Splash Screen:** Красивый виджет для рабочего стола (на базе Jetpack Glance) и плавный стартовый экран.
* 🌍 **Локализация:** Поддержка русского и английского языков.

## 🚀 Как получить API ключи для Яндекс Диска

Для работы облачной синхронизации вам потребуется Client ID от Яндекс ID.

1. Перейдите в [Кабинет разработчика Яндекс OAuth](https://oauth.yandex.ru/).
2. Нажмите **«Создать приложение»** (или «Новый клиент»).
3. Заполните название (например, `GravityMusic`) и выберите платформу **Android**.
4. В поле `ID пакета приложения (Package name)` укажите: `com.gravitymusic`.
5. В поле `Отпечаток сертификата (SHA-1)` добавьте SHA-1 ключ вашего сертификата подписи (можно получить командой `./gradlew signingReport`).
6. В разделе **Доступы** выберите `Яндекс Диск REST API`:
   * Чтение всего Диска (`cloud_api:disk.read`)
   * Запись в любом месте на Диске (`cloud_api:disk.write`)
7. Сохраните изменения. Вы получите **Client ID**.
8. Приложение перехватит авторизацию через Deep Link: `yandexauth://gravitymusic`.

## 🛠️ Как собрать APK версию

Для сборки проекта вам понадобится **Android Studio (Iguana или новее)** или установленный Gradle:

### Через консоль (терминал):
1. Откройте проект в терминале.
2. Дайте права на исполнение Gradle Wrapper:
   ```bash
   chmod +x gradlew
   ```
3. Соберите Debug APK:
   ```bash
   ./gradlew assembleDebug
   ```
   *Готовый APK будет находиться по пути: `app/build/outputs/apk/debug/app-debug.apk`*

### Через Android Studio:
1. Откройте проект.
2. Дождитесь окончания синхронизации Gradle.
3. В верхнем меню выберите **Build** > **Build Bundle(s) / APK(s)** > **Build APK(s)**.

## 📦 Описание процесса деплоя (подпись приложения)

Чтобы загрузить приложение в Google Play, NashStore, RuStore или распространять как Release, его нужно подписать ключом (Keystore).

1. В Android Studio перейдите в **Build** > **Generate Signed Bundle / APK**.
2. Выберите **APK** (или Android App Bundle для Google Play).
3. Нажмите **Create new...** под полем `Key store path` и создайте ваш `.jks` файл-ключ. Запомните или запишите пароли!
4. Заполните данные сертификата (ФИО, организация).
5. Нажмите **Next**, выберите вариант `release` и нажмите **Finish**.
6. Приложение будет подписано и оптимизировано. *Никогда не теряйте файл ключа, иначе вы не сможете выпускать обновления для этого пакета!*

> **Совет по легковесности (`R8 / ProGuard`):**  
> В файле `build.gradle.kts` для `release` сборки уже включен `isMinifyEnabled = true`. При сборке Release версии код будет автоматически ужат, а неиспользуемые ресурсы удалены, что делает приложение максимально компактным и быстрым.

## 🛑 Проверка на утечки памяти (Memory Leaks)

Во время разработки (в Debug сборках) в приложении автоматически работает **LeakCanary**. 
* Если при навигации по приложению или сворачивании плеера происходит утечка памяти (например, `Context` или `ViewModel` не уничтожились), появится уведомление с желтой иконкой птички.
* Просто нажмите на уведомление, чтобы посмотреть стектрейс проблемы.
* В **Release** версии LeakCanary полностью отключен и не влияет на производительность.
